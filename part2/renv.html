6  renv

renv is R’s package management system. Like conda for Python, it creates isolated package libraries for each project, ensuring reproducibility.

6.1 Why renv?

Without renv: - All projects share one R package library - Updating a package for one project might break another - Hard to know which packages a project needs

With renv: - Each project has its own package library - Package versions are locked in renv.lock - Anyone can recreate your exact environment

6.2 Getting Started

6.2.1 Initialize renv

In a new or existing R project:

# Install renv if needed
install.packages("renv")

# Initialize renv for this project
renv::init()

This creates: - renv/ — Local package library - renv.lock — Lockfile recording package versions - .Rprofile — Activates renv when R starts

6.2.2 The Workflow

  1. Install packages as normal with install.packages()
  2. Snapshot to record what you installed: renv::snapshot()
  3. Share your project (including renv.lock)
  4. Restore on another machine: renv::restore()

6.3 Essential Commands

6.3.1 Installing Packages

# Install from CRAN
install.packages("ggplot2")

# Install from GitHub
renv::install("tidyverse/ggplot2")

# Install specific version
renv::install("ggplot2@3.4.0")

6.3.2 Snapshot

After installing packages, save the state:

renv::snapshot()

This updates renv.lock with current package versions.

6.3.3 Restore

When you clone a project or switch machines:

renv::restore()

This installs all packages recorded in renv.lock.

6.3.4 Status

Check if your library matches the lockfile:

renv::status()

Outputs tell you if packages are: - Missing from the library - Not recorded in the lockfile - Different versions than the lockfile

6.4 The Lockfile

renv.lock is a JSON file recording every package:

{
  "R": {
    "Version": "4.3.1",
    "Repositories": [...]
  },
  "Packages": {
    "ggplot2": {
      "Package": "ggplot2",
      "Version": "3.4.2",
      "Source": "Repository"
    }
  }
}

Always commit renv.lock to Git. This is how others reproduce your environment.

6.5 What to Commit to Git

Include: - renv.lock — Package versions - renv/activate.R — Activation script - .Rprofile — Auto-activation

Exclude (in .gitignore): - renv/library/ — Actual packages (too large) - renv/staging/ — Temporary files

6.6 Updating Packages

6.6.1 Update All

renv::update()
renv::snapshot()  # Don't forget to snapshot!

6.6.2 Update Specific Package

renv::update("ggplot2")
renv::snapshot()

6.6.3 Revert an Update

If something breaks, restore the previous state:

renv::restore()

6.7 Common Issues

6.7.1 “The project is out-of-sync”

This means packages in your library don’t match renv.lock:

# See what's different
renv::status()

# Sync library to lockfile
renv::restore()

# Or update lockfile to match library
renv::snapshot()

6.7.2 Package Installation Fails

Try:

# Clear cache and reinstall
renv::install("package-name", rebuild = TRUE)

6.7.3 Starting Fresh

If things are really broken:

# Remove renv from project
renv::deactivate()
unlink("renv", recursive = TRUE)
file.remove("renv.lock")

# Reinitialize
renv::init()

6.8 Working with Bioconductor

For bioinformatics packages:

# Set up Bioconductor
renv::install("BiocManager")
BiocManager::install("DESeq2")
renv::snapshot()

renv automatically detects Bioconductor packages.

6.9 Integration with Positron

renv works automatically with Positron:

  1. Open a project with renv
  2. R automatically activates renv (via .Rprofile)
  3. Install packages as normal
  4. Run renv::snapshot() to save

The R console should show:

- Project '/path/to/project' loaded. [renv 1.0.0]

6.10 Best Practices

6.10.1 Snapshot Regularly

After installing or updating packages:

renv::snapshot()
git add renv.lock
git commit -m "Update package dependencies"

6.10.2 Check Status Before Committing

renv::status()

Ensure the lockfile reflects your actual packages.

6.10.3 Document Special Packages

If packages need special installation (GitHub, local, etc.), document it:

# Install custom packages:
# renv::install("github-user/custom-package")

6.11 Quick Reference
