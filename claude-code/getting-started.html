10  Getting Started with Claude Code

Claude Code is an AI assistant that works directly with your project files. Unlike web-based AI chat, it can read your code, edit files, run commands, and search across your codebase—all within the context of your project.

10.1 Accessing Claude Code

10.1.1 Positron Extension (Recommended)

The easiest way to use Claude Code is through the Positron extension. This keeps your code, terminal, and AI assistant in one window.

To open Claude Code in Positron:

  1. Open your project folder in Positron
  2. Click the Claude Code icon in the left sidebar (or use Cmd+Shift+P → “Claude Code: Open”)
  3. Start typing in the chat panel

The extension automatically knows which project you’re working in and can see your files.

Key shortcuts:

Action Shortcut
Submit message Enter (or Cmd+Enter depending on settings)
New conversation Click “+” or Cmd+N in the Claude panel
Reference a file Type @ and start typing the filename

10.1.2 Other Ways to Access Claude Code

You can also use Claude Code through:

  • Terminal/CLI — Run claude from your project directory (Anthropic docs)
  • Desktop app — Claude for Mac or claude.app
  • Web — claude.ai (no file access, but useful for general questions)

This guide focuses on the Positron extension since it integrates with your development workflow.

10.2 What Claude Code Can Do

Capability Examples
Understand code Explain functions, trace data flow, find patterns
Write code Create scripts, write functions, generate boilerplate
Edit files Fix bugs, refactor, add features to existing files
Run commands Execute R/Python scripts, git operations, quarto render
Search Find where things are defined, grep for patterns
Debug Trace errors, explain stack traces, suggest fixes

10.3 When to Use Claude Code

Claude Code is helpful for:

  • Setting up project structure and boilerplate
  • Debugging cryptic error messages
  • Understanding unfamiliar code in a project
  • Writing repetitive code (data loading, plotting templates)
  • Troubleshooting environment issues (conda, renv, paths)
  • Explaining what code does and how components connect

Claude Code is not a replacement for:

  • Your analytical judgment (you decide normalization methods, thresholds, interpretations)
  • Your domain expertise (you know biology, Claude knows code)
  • Careful code review (always verify outputs, especially for statistics)
  • Understanding methods you’ll use repeatedly (learn them, don’t just copy-paste)
NoteThe division of labor

You bring the scientific knowledge and analytical decisions. Claude Code brings coding speed and pattern recognition. The combination is powerful, but you’re always the one making the calls.

10.4 The Mental Model

Three concepts help you work effectively with Claude Code:

10.4.1 1. Capable but not omniscient

Claude can write sophisticated code, but it doesn’t automatically know:

  • Your specific data structures and conventions
  • Your project’s history and past decisions
  • What you’ve tried before and why it didn’t work

Solution: Tell Claude what it needs to know. Be specific about your data, your goals, and any constraints.

10.4.2 2. Context matters

Claude only knows what’s in your project folder and what you tell it. Each new conversation starts fresh—it doesn’t remember yesterday’s session.

Solution: Use project configuration files (covered in the next chapter) to give Claude persistent context about your project.

10.4.3 3. Verify, then trust

Claude often gets things right, but sometimes produces plausible-looking code with subtle bugs, especially for:

  • Statistical methods it doesn’t fully understand
  • Edge cases in your specific data
  • Domain-specific conventions it guesses at

Solution: Run the code and check the outputs. For analytical code, verify counts at each step and spot-check results.

10.5 A Quick Example

Here’s what a typical interaction looks like in Positron:

You type: > I’m getting this error when I run the script: > > Error in left_join(): ! by must be supplied when x and y have no common variables. > > The join is on line 45 of analysis.qmd

Claude responds: - Reads the relevant section of analysis.qmd - Explains that the column names don’t match between the two dataframes - Shows you the column names in each - Proposes a fix with the correct by = argument - Asks if you want to apply the edit

You review the proposed change, accept it, and the file is updated.

10.6 Reviewing Changes

Claude always shows you what it plans to do:

  • File edits — Shows a diff with the proposed changes highlighted
  • New files — Shows the full content before writing
  • Commands — Shows the command before running it

You can:

  • Accept the change
  • Ask for modifications (“Can you also add error handling for empty files?”)
  • Reject and try a different approach (“Actually, let’s do this a different way”)
